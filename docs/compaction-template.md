# Compaction Template

When summarizing a long session to preserve context through compaction, use this structure:

## 1. Primary Request

What the user originally asked for. Include exact wording if possible.

## 2. Current State

Where we are right now — what's done, what's in progress, what's blocked.

## 3. Key Decisions Made

Architectural and design choices with rationale. Include alternatives that were considered and rejected.

## 4. Files Modified

Paths and what changed in each. Include enough detail to avoid re-reading files.

## 5. Errors Encountered

What went wrong and how it was resolved. Include error messages and fixes.

## 6. Important Discoveries

Non-obvious things learned during the session — project conventions, gotchas, dependency quirks, undocumented behavior.

## 7. Pending Tasks

What still needs to be done, in order. Include enough detail to continue without re-reading the conversation.

## 8. User Preferences

Any preferences expressed during the session — style, approach, constraints, explicit instructions.

## 9. Context to Preserve

Specific details that would be expensive to re-discover: file paths, line numbers, API signatures, configuration values, test commands.
